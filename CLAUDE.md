# 퍼포먼스 마케팅 데이터 분석 프로젝트

## 업무 개요
매일 채널 데이터(노출/클릭/비용)와 앱스플라이어 데이터(전환/매출)를 업로드하고,
두 데이터를 조인·전처리하여 광고 성과 인사이트를 도출하는 작업.

---

## 폴더 구조

```
0626클로드코드/
├── CLAUDE.md               ← 이 파일
├── dashboard.py            ← Streamlit 대시보드
└── data/
    ├── channel/            ← 채널 로우 데이터 (매일 추가)
    │   └── YYYY-MM-DD_channel.parquet
    └── appsflyer/          ← 앱스플라이어 로우 데이터 (매일 추가)
        └── YYYY-MM-DD_appsflyer.parquet
```

**파일 추가 규칙:**
- 파일명: `YYYY-MM-DD_channel.parquet` / `YYYY-MM-DD_appsflyer.parquet`
- 업로드 주기: 매일 전날 데이터 (D+1 적재)
- 수정 시: 같은 파일명으로 덮어쓰기
- 누락일: 파일 없으면 해당 날짜는 집계에서 제외 (오류 아님)

## 소재 네이밍 컨벤션

**형식:** `{소재타입}_{카테고리}_{시즌}_{AB테스트}_{버전}`

| 구성요소 | 값 | 설명 |
|---------|-----|------|
| 소재타입 | VID | 동영상 |
| | IMG | 이미지 (단일) |
| | CRS | 캐러셀 |
| | TXT | 텍스트 (검색광고) |
| 카테고리 | 플러스멤버십 | 멤버십 가입 소구 |
| | 적립혜택 | 포인트/적립 혜택 소구 |
| | 할인쿠폰 | 할인 쿠폰 소구 |
| | 배송혜택 | 배송 혜택 소구 |
| | 특가 | 특가 상품 소구 |
| | 신상품 | 신상품 소구 |
| 시즌 | 겨울 / 봄 / 여름 / 가을 | 시즌 구분 |
| AB테스트 | A / B / (없음) | AB테스트 구분. 없으면 단독 소재 |
| 버전 | v1 / v2 / ... | 소재 수정 버전 |

**예시:**
- `VID_플러스멤버십_겨울_A_v1` → 동영상, 멤버십 소구, 겨울 시즌, A안, 첫 번째 버전
- `IMG_적립혜택_겨울_v2` → 이미지, 적립혜택 소구, 겨울 시즌, AB테스트 없음, 두 번째 버전
- `CRS_할인쿠폰_겨울_v1` → 캐러셀, 할인쿠폰 소구, 단독 소재

## 캠페인 구조

**캠페인 ID 형식:** `{채널prefix}_CMP_{번호}_{목적}`

| 캠페인 | 채널 | 목적 | 타겟 그룹 |
|--------|------|------|----------|
| GGL_CMP_01_플러스가입 | 구글 | 신규 멤버십 가입 | 논타겟, 유사타겟 |
| GGL_CMP_02_리타겟팅 | 구글 | 기존 방문자 재유입 | 리마케팅 |
| GGL_CMP_03_첫구매 | 구글 | 비구매 회원 전환 | 논타겟, 유사타겟 |
| META_CMP_01_신규유저 | 메타 | 신규 유저 획득 | 논타겟 |
| META_CMP_02_룩얼라이크 | 메타 | 유사 타겟 확장 | 유사타겟 |
| META_CMP_03_재구매 | 메타 | 기존 고객 재구매 | VIP, 윈백 |
| NVR_CMP_01_브랜드KW | 네이버 | 브랜드 키워드 검색 | 논타겟 |
| NVR_CMP_02_일반KW | 네이버 | 일반 키워드 검색 | 논타겟 |
| NVR_CMP_03_재구매 | 네이버 | 기존 고객 재구매 | VIP, 윈백 |

**타겟 그룹 정의:**
- **논타겟**: 관심사 기반 일반 타겟팅
- **유사타겟**: 기존 고객 기반 룩얼라이크
- **리마케팅**: 사이트/앱 방문자 재타겟
- **VIP**: 고가치 기존 구매 고객
- **윈백**: 이탈 고객 재활성화

## 지표 계산 공식

> **기준 데이터: 채널 데이터** (노출·비용은 채널 기준, 전환도 채널 기준 사용)
> AF 데이터는 크로스체크 및 어트리뷰션 검증용

| 지표 | 공식 | 단위 |
|------|------|------|
| CTR | 클릭 / 노출 × 100 | % |
| CVR | 구매 / 클릭 × 100 | % |
| ROAS | 구매매출 / 비용 × 100 | % |
| CPA | 비용 / 구매 | 원/건 |
| CPC | 비용 / 클릭 | 원/회 |
| CPM | 비용 / 노출 × 1000 | 원/1000회 |

**채널 vs AF 숫자 차이 이유:**
- 채널 데이터: 광고 플랫폼 자체 집계 (라스트클릭 기준)
- AF 데이터: 앱스플라이어 어트리뷰션 (중복 제거, 뷰스루 포함 가능)
- 차이가 20% 이상이면 어트리뷰션 설정 확인 필요

## 분석 규칙

**집계 기준:**
- 기본 단위: 채널 > 캠페인 > 그룹 > 소재
- 기간: 일별 집계, 주간/월간 롤업
- 비교 기준: 전주 대비 (WoW), 전월 대비 (MoM)

**이상치 기준 (플래그 조건):**
- CTR > 15% → 비정상 트래픽 의심
- ROAS < 100% → 적자 캠페인
- 비용 = 0 & 노출 > 0 → 데이터 누락 의심
- 채널 구매 vs AF 구매 차이 > 30% → 어트리뷰션 이슈

**소재 분석 시:**
- 소재타입(VID/IMG/CRS/TXT)별 성과는 반드시 분리 비교
- AB테스트 소재는 A/B를 쌍으로 비교 (단독 비교 금지)
- 노출 < 1,000 소재는 통계적 유의성 없음 — 판단 보류

## 자주 쓰는 분석 패턴

```python
# 채널+AF 조인
joined = pd.merge(channel_df, af_df,
                  on=['일', '캠페인', '그룹', '소재'],
                  how='left',
                  suffixes=('_ch', '_af'))

# 소재타입 추출
df['소재타입'] = df['소재'].str.extract(r'^(VID|IMG|CRS|TXT)_')

# AB테스트 구분
df['AB'] = df['소재'].str.extract(r'_(A|B)_v\d+$')
df['AB'] = df['AB'].fillna('단독')
```
